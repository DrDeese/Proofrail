# Proofrail project status

## Maturity

**Internal Alpha**

Proofrail is suitable for controlled internal repositories, technically capable design partners, and read-only CI evaluation. It is not yet suitable as a turnkey hosted platform, a universal behavioral verifier, or a supported general-availability product.

## Current proven capabilities

- Exact Git-range inspection.
- Deterministic claim drafting.
- Claim freshness checks.
- Artifact-level verification.
- Acceptance-policy evaluation.
- A local GitHub Action.
- Read-only workflow permissions.
- Deterministic preflight.
- Structured scope escalation.

## Current limitations

- No package distribution.
- No formal release or compatibility policy.
- No hosted service.
- Limited external-repository evidence.
- Fork-PR behavior has not been independently exercised.
- No authentication of authorship or timestamps.
- No proof of deployment, browser rendering, external systems, or human intent.
- No formal support or incident-response commitment.

These limitations are product boundaries, not transient implementation details. A completed artifact-level check must not be represented as proof of an external behavioral outcome.
